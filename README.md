# PainMap
# 篮球运动员足踝伤病智能筛查系统

一个纯前端（HTML / CSS / 原生 JavaScript）的足踝伤病筛查工具，
根据《Basketball Foot & Ankle Injury Screening App — Master Specification》
Part 1–5 实现完整流程：红旗排查 → 急性/慢性分流 → 病史问卷 → 疼痛定位地图 →
附加症状 → 篮球功能测试 → 既往伤病史 → 评分引擎（含全局规则与优先覆盖规则）→
Top 3 排序报告（含匹配度、支持因素、推荐特殊检查）。

**⚠️ 重要提示：本工具仅用于初步风险筛查与自测引导，不构成正式医学诊断，
不能替代医生检查或影像学检查。**

---

## 快速开始

不需要安装任何依赖、不需要构建工具。

**方式一：直接打开**
双击 `index.html`，用浏览器打开即可（部分浏览器出于安全策略可能限制本地
文件间的资源加载，若样式或脚本无法加载，请使用方式二）。

**方式二：本地起一个静态服务器（推荐）**
```bash
# Python 3
cd hoopfoot
python3 -m http.server 8080
# 然后浏览器打开 http://localhost:8080
```
或者用 Node：
```bash
npx serve .
```

## 本次更新（Rehab + 可调权重）

1. **既往伤病史 → 乘法权重**：拖动条 0–10（记为 n），选中的既往伤病分数变为
   `score × (1 + 0.1 × n)`（n=1 → ×1.1，n=5 → ×1.5，n=10 → ×2.0），替换掉原来
   的加法逻辑。
2. **Rehab 建议**：`js/data/rehab.js` 中的 `REHAB` 数据库为每个伤病提供分阶段
   的康复 / 训练调整建议，结果页 Top3 每张卡片下方新增"康复 / 训练调整建议"板块。
   没有专属数据的伤病会退回 `REHAB_FALLBACK` 通用建议（目前所有可见伤病均已覆盖）。
3. **特殊检查阳性 / 阴性再评分（含详细操作说明）**：在既往病史之后新增"特殊检查"页面
   （`showSpecialTestScreen`），数据来自 `js/data/specialTests.js`（`SPECIAL_TESTS`），
   针对初步 Top3 的每一项推荐特殊检查，展示：
   - **怎么做**：分步骤操作说明
   - **阳性表现（怎样算阳性）**：具体的疼痛 / 异常表现描述
   - **阴性表现（怎样算阴性）**：与健侧对比无明显差异、未复现上述表现
   - 部分检查标注"金标准"徽章，并附敏感性 / 特异性数据（供参考）
   
   用户为每项检查选择 阳性 / 阴性 / 未测试，并配两条独立拖动条（0–10）：
   - 阳性权重 p → 命中项 `score × (1 + 0.1 × p)`
   - 阴性权重 n → 命中项 `score × (1 − 0.1 × n)`
   
   调整后重新排序生成最终报告，结果卡片会注明具体应用了几次阳性 / 阴性调整。
   暂无收录检查的伤病（如 Morton 神经瘤、CECS）会退回 `SPECIAL_TEST_FALLBACK`
   通用触诊 + 功能动作激发说明。

4. **加分透明度页面**：既往病史之后、特殊检查之前，新增"所有有分数的伤病"页面
   （`showScoreBreakdown`），不再局限于 Top3，而是列出**所有分数 > 0 的伤病**，
   并且每个伤病下面逐条列出具体是"哪道题 = 哪个选项"贡献了多少分，来源用颜色
   区分：
   - 灰色：问卷基础题（受伤机制、疼痛位置、疼痛形态等）
   - 橙色：全局规则（活动后疼痛变化、肿胀时相、功能测试结果、瘀青等）
   - 青色：既往病史加权倍数
   - 红色：覆盖规则（Override，如"无法负重+骨点压痛→骨折 +100"）

## 评分引擎说明

- 每个伤病的基础分由多组 `rule: { points, when: [...] }` 构成，`when` 内的
  条件为“与”关系，条件之间用命中的题目字段（如 `mechanism`、`primary_location`、
  `pain_shape` 等）匹配用户作答。
- **全局规则**（活动后疼痛变化、持续性疼痛、诱发动作、肿胀时相、单脚站立 /
  单脚跳 / 变向 / 蹬地功能测试、瘀青、发热、既往反复崴脚等）在基础评分之后叠加，
  与规格书 Part 4 一致。
- **既往伤病史**按用户设置的 0–10 权重滑块，为其历史上患过的伤病直接加分。
- **覆盖规则（Override）**优先级最高，在所有评分结束后执行：
  - 明显畸形 → 直接判定疑似脱位，终止评估，建议立即就医（红旗规则同样含此逻辑）
  - 无法负重 + 骨点压痛 → 四类骨折分数各 +100
  - 中足痛 + 旋转机制 → Lisfranc 分数 +100
  - 踝上方疼痛 → 高位踝扭伤分数 +100
- 最终按分数降序输出 Top 3，并以 `Top1 分数` 为基准计算匹配百分比。

## 关于疼痛定位地图

规格书原文要求“可旋转 360° 的 3D 足部模型”。为了保证在任意设备（含移动端）
无需额外 3D 引擎即可稳定运行，这里用了**三个可切换视图的简化 2D 示意图**
（背侧/内外侧、后方、足底）来实现同样的“先选大区域，再选具体位置”交互。
如果需要更精细的解剖学 3D 模型，可以将 `js/painmap.js` 中的 SVG 替换为
Three.js / model-viewer 等方案，只需保持 `data-region` 点击回调接口一致即可。

## 二次开发提示

- 所有中文问题文案、选项、评分权重都集中在 `js/data/` 目录下，便于医学顾问
  审核和调整分值，无需改动界面逻辑。
- 如需新增伤病，只需在 `acuteConditions.js` / `chronicConditions.js` 中新增
  一个条目（`nameZh`、`nameEn`、`category`、`rules`），评分引擎会自动纳入排序。
- 如需接入真实医生审核流程或后端存档，可以在 `showResults()`（`js/app.js`）
  末尾追加一次 API 提交。
